# Output Template

Use this skeleton when the user did not ask for a custom format.

## 1. Search brief

- Problem:
- Target stack:
- Runtime:
- Deployment model:
- License sensitivity:
- Reusable target:

## 2. Candidate comparison table

Minimum columns:

| Repo | Why relevant | Stack fit | Stars | Last activity date | License | Reusable surface | Risks |
| --- | --- | --- | ---: | --- | --- | --- | --- |

Use absolute dates.

## 3. Top repos to clone

- List up to 3 repos.
- Give one short reason for each.
- If no repo is worth cloning, say that explicitly.

## 4. Local analysis summary

- entrypoints
- reusable modules or patterns
- tightly coupled parts to avoid

## 5. Reuse plan

- what to reuse
- source repo and file attribution
- expected adaptation effort

## 6. Risks and exclusions

- license notes
- integration risks
- what was intentionally not reused
